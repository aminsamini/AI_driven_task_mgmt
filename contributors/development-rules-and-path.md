## Development Guidelines
1. **Code Style**: Follow standard Python (PEP 8) and project-specific conventions.
2. **Git Workflow**: Use feature branches and pull requests.
3. **Documentation**: Update documentation for any new features or changes.

## Key Considerations for Contributors
- Ensure all agents are designed with the "helpful assistant" persona in mind (as seen in `main.py`).
- Verify API keys and environment variables are set up correctly.

## Project Workflow
1. **Planning**: Review requirements and create a plan.
2. **Implementation**: Write code and tests.
3. **Verification**: Run tests and verify functionality.
4. **Review**: Submit for review.
